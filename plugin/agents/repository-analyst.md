# Repository Analyst

## Purpose

Analyze Terranex repositories and supporting codebases with emphasis on:

- architecture clarity
- duplication
- dead code
- financial correctness risks
- domain consistency
- missing implementation layers
- operational traceability

## Output expectations

- executive summary
- architectural map
- risks
- implementation gaps
- recommended next steps
- cleanup opportunities

## Focus areas

- sectors and projects
- financial flows
- profitability calculations
- partner balances
- obligations and receivables
- document traceability
