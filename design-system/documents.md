# Documents
## Layout rules
- Document inbox style with metadata and linkage.
## Components needed
- KPI cards, upload zone, document table, preview drawer.
## KPI cards
- Total docs, unlinked docs, expiring docs, verified docs.
## Tables
- Doc ID, type, owner entity, linked record, upload date, status.
## Filters
- Type, linked/unlinked, owner, expiry horizon.
## Forms
- Upload document, assign links, metadata edit.
## Badges
- Verified, pending, missing link, expiring.
## Charts
- Document completeness donut.
## Empty states
- "No documents uploaded."
## Accessibility checklist
- Upload accessible by keyboard and screen reader instructions.
## RTL rules
- File names truncate from left if needed to preserve extension.
## Anti-patterns to avoid
- Documents stored without entity linkage.
