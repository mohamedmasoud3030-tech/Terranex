# Settings
## Layout rules
- Category sidebar + details pane.
## Components needed
- Role matrix, preference forms, audit settings log.
## KPI cards
- Active users, role profiles, pending approvals, policy exceptions.
## Tables
- Users, roles, last login, access scope, status.
## Filters
- Role, status, last login range.
## Forms
- User invite, permission template, notification policy.
## Badges
- Admin, finance, operations, viewer, suspended.
## Charts
- Optional access activity chart.
## Empty states
- "No users configured."
## Accessibility checklist
- Toggle labels explicit, permission matrix keyboard friendly.
## RTL rules
- Role columns maintain readable density.
## Anti-patterns to avoid
- Hidden permission implications.
